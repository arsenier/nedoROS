Мне необходимо написать сценарий для видео, рассказывающем о нашем ROS2 роботе.

В видео должны быть освещены темы:

обзор всей системы в целом

  • робот:
  • микроконтроллер (Arduino Uno)
  • одноплатник (Raspberry Pi 4b)
  • сервер навигации
  • небесная веб камера
  • классификатор уточек
  • система глобального позиционирования

аппаратная часть

  • механика
  • фанерные детали
  • 3д печатные детали
  • исполнительные механизмы (моторы, сервоприводы)
  • датчики (усики, лидар, бортовая веб камера)
  • электроника
  • схема подключения
  • включение аварийной кнопки

система управления

Модифицированная 3х уровневая архитектура с элементами архитектуры подчинения (subsumption)

программа микроконтроллера

  • уровень контроллер (controller)
  • главный цикл
  • парадигма ITSPA
  • контроль скорости моторов
  • программирование движения захвата через параметрические функции
  • единственная задача: качественно отрабатывать присланное задание и давать обратную связь
  • драйвер для ROS2 на бортовом компьютере

программа бортового компьютера

  • уровень секвенсор (sequencer)
  • основная логика управления движением и принятия решений
  • парадигма ITSPA
  • топики это датчики и исполнительные узлы
  • набор элементарных поведений для выполнения отдельных задач
  • для всех поведений блоки TS и A обеспечиваются использованием ROS
  • блоки P для каждого поведения выделены в отдельные циклы
  • для каждого поведения написан свой цикл, это поведение выполняющий
  • алгоритм выполнения задачи прописывается в виде последовательности элементарных поведений, с использованием стандартных функций управления потоком исполнения программы

программа системы GPS

  • уровень планировщик (deliberator)
  • алгоритмы видеозрения
  • распознавание charuco доски и aruco меток
  • определение глобальных координат робота
  • классификация уточек по типу

На основании списка тем с основными мыслями напиши черновик сценария в виде более подробного списка тем и главных моментов, которые необходимо осветить. Напиши заготовки текста для каждой части, укажи места где необходимо добавить детали. Общий хронометраж видео - 30 минут. Для каждой части укажи примерное экранное время.

Черновик сценария видео: «ROBOT-2025: наш ROS2-робот с трёхуровневой архитектурой»

Общий хронометраж: 30 минут
Формат: экранное время разделено между ведущим (или закадровый голос) + натурные съёмки робота, схемы, 3D-модели, фрагменты кода.

1. Вступление (1 мин)

Текст:

«Привет! Сегодня мы детально разберём нашего робота на ROS2. Он умеет ориентироваться по глобальным координатам, распознавать уток, автономно объезжать препятствия и выполнять захват. В основе — модифицированная трёхуровневая архитектура с элементами subsumption. За 30 минут пройдём путь от фанерной механики до алгоритмов видеозрения. Поехали!»

Что на экране:
- Красивое общее видео робота, выполняющего задачу.
- Логотип проекта.

2. Обзор всей системы (3 мин)

2.1. Робот целиком (1 мин)

Текст:

«Наш робот — это гибрид Arduino Uno и Raspberry Pi 4B. Arduino отвечает за низкоуровневый контроль моторов и датчиков, Raspberry — за логику поведения. Связь через ROS2: драйвер на Pi отправляет команды и получает телеметрию.»

Детали, которые нужно показать:
- Фото/видео робота, указать где Arduino, где Pi.
- Схема: Arduino ↔ (USB/serial) ↔ Raspberry Pi.

2.2. Сервер навигации (2 мин)

Текст:

«Верхний уровень — сервер навигации, который работает на отдельном ПК или прямо на Raspberry, если хватает ресурсов. В его задачи входит:
- Небесная веб-камера (смотрит на потолок с ArUco-маркерами для глобальной локализации).
- Классификатор уточек — нейросеть, различающая, например, резиновых, деревянных, живых.
- Система глобального позиционирования — объединяет данные с камеры и, при необходимости, GPS-модуля (на улице).»

Детали: вставить скриншот работы детектора ArUco, пример классификации уточек.
Временная метка: с 1:00 по 4:00.

3. Аппаратная часть (6 мин)

3.1. Механика (2 мин)

Текст:

«Корпус — лазерная резка фанеры. Все кронштейны, шестерни и захваты напечатаны на 3D-принтере. Исполнительные механизмы: два DC мотора с энкодерами для колёс, сервоприводы для захвата и поворота камеры.
Датчики:
- Усики (тактильные концевые выключатели) — для обнаружения столкновений.
- Лидар (например, YDLIDAR X2) — для построения локальной карты.
- Бортовая веб-камера — для распознавания объектов спереди.»

Показать: крупные планы фанерных деталей, 3D-печатных узлов, работу захвата, монтаж лидара.

3.2. Электроника (2 мин)

Текст:

«Электроника построена на двух платах:
- Arduino Uno: читает энкодеры, управляет моторами через драйвер L298N (или аналоги), опрашивает усики и сервоприводы.
- Raspberry Pi 4B: принимает команды от Arduino, обрабатывает лидар, камеры, общается с сервером навигации.
- Схема подключения: Pi получает питание 5В/3А, Arduino — от отдельного регулятора. Лидар через USB на Pi.»

Текст про аварийную кнопку:

«Аварийная кнопка — разрыв цепи питания моторов, подключена к входу Arduino. При нажатии — мгновенная остановка, даже если ROS завис.»

Показать: схему на экране, реальную кнопку на роботе.
Детали: добавить фото распиновки, если нужно.

3.3. Демонстрация аппаратной части (2 мин)

Текст:

«Давайте посмотрим, как это работает вместе. Сейчас я нажимаю кнопку — моторы останавливаются. Проверим усики — при касании робот отъезжает назад.»

Видео: ручное тестирование кнопки, реакции усиков, движение по команде с пульта.

4. Система управления (16 мин)

4.1. Общая архитектура (1 мин)

Текст:

«У нас модифицированная трёхуровневая архитектура с элементами subsumption.
- Уровень Controller (Arduino) — выполнение примитивных команд, ПИД-регуляторы скорости.
- Уровень Sequencer (Raspberry Pi) — выбор поведения на основе датчиков и целей.
- Уровень Deliberator (сервер GPS) — глобальное планирование и распознавание.
Subsumption проявляется в том, что более приоритетное поведение (например, „избегание столкновения“) может вытеснять текущее.»

Схема: три блока со стрелками, указать потоки данных.

4.2. Программа микроконтроллера (Controller) (5 мин)

Текст:

«Arduino работает по парадигме ITSPA (Iterative, Timed, Sense-Plan-Act). Главный цикл:
- Считать датчики (энкодеры, усики, кнопку).
- Получить задание из ROS2 (через serial).
- Вычислить управляющие сигналы для моторов (PID).
- Выполнить движение захвата по параметрическим функциям (например, синусоидальная траектория для обхода помехи).
- Отправить обратную связь: текущая скорость, положение, сработали ли усики.
Единственная задача Arduino — качественно отрабатывать присланное задание и не думать о глобальной цели.»

Демо:
- Фрагмент кода: void loop() { sense(); plan(); act(); }
- Параметрическая функция для захвата — график на экране.
- Ролик: робот выполняет команду «прямо 1 метр» — Arduino стабилизирует скорость.

4.3. Программа бортового компьютера (Sequencer) (5 мин)

Текст:

«На Raspberry Pi запущен ROS2-узел — “секвенсор”. Он тоже следует ITSPA, но на уровне поведения.
- Топики ROS2 — это датчики (/lidar, /camera, /bumpers) и исполнители (/cmd_vel, /gripper).
- Для каждого элементарного поведения („ехать вдоль стены“, „объехать утку“, „захватить“) написан отдельный цикл.
- Блоки TS (Time & Sense) и A (Act) обеспечиваются самим ROS2: подписка на топики и публикация команд.
- Блоки P (Plan) — внутри каждого поведения свой цикл, где на основе сенсоров решается, что делать прямо сейчас.
- Алгоритм выполнения задачи — это последовательность вызовов поведений с обычными if/while и ожиданием событий. Например:
nav_to(x,y); detect_duck(); grasp();»

Детали: показать код на экране (C++/Python), как секвенсор переключает поведения.
Видео: робот объезжает препятствие — наложение текста, какое поведение активно.

4.4. Программа системы GPS (Deliberator) (4 мин)

Текст:

«Верхний уровень — планировщик. Он запущен на мощном ПК или на Raspberry, но с выносом тяжёлых сетей. Задачи:
- Распознавать Charuco-доску и ArUco-метки на потолке (глобальная локализация).
- Вычислять глобальные координаты робота (x, y, угол).
- Классифицировать уток по типу: резиновая, деревянная, живая — с помощью свёрточной нейросети.
Планировщик посылает секвенсору цель „переместиться в точку A“ или „утка типа X находится в (x,y)“.
Здесь же реализован глобальный планировщик пути (A* на карте).»

Демо:
- Работа ArUco-детектора на видео с потолочной камеры.
- Окно классификации: наезд камеры на утку, вывод типа.
- Rviz с глобальным планом и траекторией робота.

4.5. Взаимодействие уровней на примере (1 мин)

Текст:

«Пример: планировщик даёт секвенсору команду „подъехать к утке типа A“. Секвенсор активирует поведение „навигация к точке“. Поведение отправляет Arduino цели скорости. Если лидар видит препятствие — секвенсор переключается на поведение объезда (subsumption: более приоритетное). Arduino продолжает выполнять локальные команды, не зная про утку. Как только препятствие позади — возврат к исходному поведению.»

Анимация: блок-схема с изменением активных уровней.

5. Заключение (3 мин)

Текст:

«Мы показали:
- Как собрать робота из фанеры и 3D-печати.
- Почему мы разделили управление на три уровня.
- Как Arduino через ROS2 отрабатывает примитивы, а Raspberry решает, что делать.
- Как сервер навигации даёт глобальные цели.
Такой подход делает робота надёжным (аварийная кнопка и усики всегда работают), гибким (добавление новых поведений — просто написать ещё один цикл) и умным (глобальное планирование с распознаванием).

Благодарность:

«Спасибо за внимание! Если хотите увидеть исходники или сборку — ссылки в описании. Подписывайтесь, в следующем видео разберём код секвенсора и калибровку ArUco.»

На экране:
- Контактная информация, ссылки на GitHub, титры.
